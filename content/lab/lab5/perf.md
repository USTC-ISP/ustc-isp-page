## 1. 消除循环中的重复计算

**核心思想**：循环内部若包含每次迭代结果均一致的计算逻辑，则该计算被称为“循环不变式”，将此类计算移至循环外部可减少冗余。

### 案例分析
考虑以下求和代码：
```c
int sum(int a[]) {
    int s = 0;
    for (int i = 0; i < length(a); i++) { // 每次都会调用 length()
        s += a[i];
    }
    return s;
}
```
由于数组 `a` 的长度在循环中是固定不变的，每次迭代都调用 `length(a)`（如果是字符串 `strlen` 更是 $O(n)$ 复杂度）会造成严重的性能浪费。

### 优化方案
```c
int sum(int a[]) {
    int s = 0;
    int len = length(a); // 提前提取到循环外
    for (int i = 0; i < len; i++) {
        s += a[i];
    }
    return s;
}
```
**本实验应用**：在卷积操作中，卷积核的大小（`K_SIZE`）和偏移量（`K_RADIUS`）应当在最外层计算好，不要在最内层像素点计算时重复执行加减法逻辑。


## 2. 寄存器变量

**核心思想**：内存访问（Load/Store）比寄存器操作慢得多。尽量使用局部变量暂存累加结果。

### 案例分析
```c
void mult(int a[], int *dst) {
    int len = length(a);
    for (int i = 0; i < len; i++) {
        *dst = *dst * a[i]; // 每次迭代都要读写一次内存
    }
}
```
CPU 无法确定指针 `dst` 是否可能指向数组 `a` 中的某个元素（内存别名风险），因此每次都会老老实实地从内存读出 `*dst`，计算后再写回内存。

### 优化方案
通过引入局部变量（通常会被分配至寄存器）暂存中间计算结果。
```c
void mult(int a[], int *dst) {
    int len = length(a);
    int temp = *dst; // 暂存到寄存器
    for (int i = 0; i < len; i++) {
        temp = temp * a[i];
    }
    *dst = temp; // 最终写回
}
```

**本实验应用**：在计算某个像素的卷积和时，使用一个 `float sum = 0.0f;` 局部变量累加，等 Kernel 的乘法全部完成后，再把结果写回 `dst[i*n + j]`。

## 3. 循环展开与指令级并行

**核心思想**：通过增加单次迭代中计算元素的数量，减少循环控制开销（索引自增、条件判断），并帮助 CPU 填满计算流水线。

### 硬件机制

现代 CPU 具有深度流水线。如果后续指令不依赖于前续指令的结果，CPU 可以同时发射多条指令。展开循环可以：
+ 减少开销：降低了分支指令在总指令数中的占比。
+ 隐藏延迟：浮点乘加指令（FMA）通常具有 3-5 个周期的延迟。通过展开循环并使用多个独立的累加器，可以利用 CPU 的乱序执行特性填充这些延迟周期。

### 优化示例
针对 $3 \times 3$ 的固定卷积核，不要写 `for(ki=0;...){ for(kj=0;...){...} }`，而是直接平铺开：
```c
// 手动展开
sum += p[0] * k00 + p[1] * k01 + p[2] * k02 +
       p[n] * k10 + p[n+1] * k11 + p[n+2] * k12 +
       p[2n] * k20 + p[2n+1] * k21 + p[2n+2] * k22;
```

## 4. 局部性原理与分块技术

本题的图像处理操作是典型的**内存密集型** 计算任务 。优化此类程序的关键不在于减少算术运算，而在于如何高效地调度数据在**存储器层级结构** 中的流动，最大化**缓存命中率** 。

### 局部性原理

程序在执行时，所访问的地址往往呈现出聚集特征，这被称为局部性原理。它分为两个维度：

* **时间局部性**：如果一个内存位置被引用，那么它在短时间内极有可能被再次引用。
    * *实验应用*：在卷积计算中，卷积核的系数会被重复读取 $N^2$ 次，应当通过寄存器或高阶缓存将其保留。
* **空间局部性**：如果一个内存位置被引用，那么其邻近的地址在短时间内极有可能被访问。
    * *实验应用*：按行访问图像数据（行主序）比按列访问更符合空间局部性，能显著提升缓存命中率。

### 缓存行与访存步长

硬件层面的数据交换并非以字节为单位，而是以**缓存行 (Cache Line)** 为基本单位（在现代处理器上通常为 **64-byte**）。

1.  **缓存预取**：当 CPU 访问地址 $A$ 时，硬件会自动将包含 $A$ 在内的整条缓存行（连续 16 个 `float` 元素）一次性加载至 L1 Cache。
2.  **步长影响**：
    * **步长为 1**：按行顺序访问，每加载一次缓存行可供后续 15 次访问直接命中（Hit），效率最高。
    * **大步长访问**：例如按列访问（跨行跳转），每次访问都会产生一次**缓存失效 (Cache Miss)**，迫使 CPU 停工等待内存控制器从高延迟的 DRAM 中搬运数据。


### 分块技术

当图像维度 $N$ 较大（如 $N \ge 1024$）时，由于单行数据量可能已超过 L1/L2 Cache 的容量，传统的线性扫描仍会导致缓存行在被重用前就被踢出。**分块 (Blocking)** 是一种针对性技术，旨在将大矩阵划分为子块，使每个小块的数据能完全驻留在缓存中 。

* **原理**：通过将多重循环改造成“块循环 + 块内循环”，强制处理器在完成当前小块的所有计算（包括所有卷积邻域的重用）后再移动到下一块。
* **优势**：该技术将数据重用的范围从“行”缩小到了“块”，显著降低了对主存带宽的压力 。

## 5. 分支消除
现代处理器的流水线执行高度依赖分支预测器。一旦预测失败，处理器必须丢弃已进入流水线的后续指令，代价极高。我们需尽量减少热点代码中的条件分支，尤其是那些方向不确定的分支。

**本实验应用**：

+ 在卷积最内层循环中实施边界检查 `if (row >= 0 && row < n ...)` 会引入大量方向不确定的条件分支，严重干扰指令预取与执行。
+ 优化方案：通过在原始图像外围填充一层宽度等于卷积半径的空白区域，可以将复杂的边界处理逻辑从热点路径中剥离。将计算核心转化为无分支的线性代码流。
